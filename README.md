# Tema 7 · Detección de las necesidades del cliente

Repositorio estático preparado para GitHub Pages.

## Incluye
- estética tipo aventura gráfica
- fase previa para mapa conceptual manuscrito
- 6 fases con los apartados solicitados
- prácticas ampliadas por apartados
- entrega manuscrita obligatoria en todas las prácticas
- tests variables por fase con banco amplio de preguntas
- 10 preguntas por intento, respuestas mezcladas y explicaciones
- elementos visuales en preguntas: diagramas e imágenes SVG
- guardado automático del progreso en `localStorage`
- panel docente oculto con `Alt + Shift + D`
- código docente con formato `mmddaa`
- certificado final imprimible

## Publicación en GitHub
1. Sube el contenido completo del repositorio.
2. Mantén `index.html` en la raíz.
3. Activa GitHub Pages desde la rama principal.

## Estructura
- `index.html`
- `certificado.html`
- `css/styles.css`
- `js/data.js`
- `js/app.js`
- `assets/adventure-map.svg`

## Nota
El progreso se guarda en el navegador del dispositivo del estudiante. Si cambia de navegador o borra los datos locales, ese progreso se perderá.
